# 良禽佳木标准包失败样例类型

这份清单用于记录“什么叫明显失真”，帮助不同工具在回归时快速识别错误方向。

## 常见失败类型

- 丢 logo：最终成品没有使用真实品牌标志，或拿纯文字品牌名代替
- 图标粗糙：图标线条、比例、风格不统一，破坏整体工艺感
- 渠道越权：因为换成长图、H5、卡片，就切成另一套品牌风格
- 中文退化：按钮、标签、标题为了极简而缩成英文式短词
- 元信息泄漏：成品里出现 `Pack`、`Preview`、`Page Goal` 等系统说明词
- 促销化：品牌页或报价卡滑向电商促销页、海报页、报表截图
- 错误承载：把 PPT、报价卡、H5、原型、设计画布或动效统一套成网页落地页
- 无上下文高保真：未读取 token、组件、资产或参考结构，就凭印象生成泛型山寨稿
- 内容注水：用假数据、无意义数字、假标签、假卖点或多余图标填满版面
- 低质仿造：缺少真实素材时硬画 SVG 图像、粗糙图标或伪 logo，而不是使用 placeholder 或停在 spec
- 变体漂移：把 `design-packs/` 当作自由换风格，而不是受控 tweak 空间

## 记录建议

每次出现失败样例，建议至少补 4 项：

1. 触发题
2. 实际输出失真点
3. 应触发但未触发的 guardrail
4. 后续应补在哪一层修复

优先记录为：

- `DESIGN.md` 约束缺失
- `artifact-surfaces/` 载体约束缺失
- `assets/brand/` 资产阻断缺失
- `evaluation/` 回归题覆盖不足
- `design-packs/` tweak 轴或锁定轴不明确
- 具体适配器执行偏差
