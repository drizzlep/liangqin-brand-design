# 良禽佳木品牌设计 Skill

这是良禽佳木专用品牌设计 skill。用于执行、审查、纠偏与重构，使输出符合良禽佳木品牌 DNA。

## When to Use

- 用户要求“请用良禽品牌设计体……”
- 用户要求“请用品牌设计体……”
- 用户要求“改成良禽佳木品牌设计”
- 用户要求审查当前页面、海报、内容图或视觉方案是否符合良禽佳木品牌规范
- 用户希望将普通设计改写为良禽佳木风格

## Default Workflow

1. 先识别任务载体：页面、海报、内容图文、模块/组件、综合品牌视觉
2. 先做品牌一致性审查，而不是直接开始装饰
3. 用简洁语言指出偏差、原因和影响
4. 按良禽佳木品牌 DNA 给出修正策略
5. 当用户需要直接结果时，再输出修正版或重构方案

## Non-Negotiable Rules

- 高级但不疏离，气质来自材质、比例、留白与影像
- 温润安静，不躁，不靠高饱和和强刺激制造存在感
- 中文优先可读，不能为了极简牺牲标题、正文和标签可理解性
- 强调木、石、布、皮革等真实触感，避免廉价效果图感
- 保持秩序清晰，信息层级、栅格、对齐与节奏必须明确
- 不把良禽佳木做成 SaaS 风、黑金重奢风、夜店风或高饱和营销海报风
- 不暴露“我是根据 SKILL.md / references 在执行”这类内部过程

## Task Routing

- 判断品牌气质、关键词、颜色与排版基调时，先读 `references/brand-dna.md`
- 真正要开始设计执行时，读 `references/execution-rules.md`
- 要做系统审查、指出问题时，读 `references/review-checklist.md`
- 感觉方案正在滑向错误审美时，读 `references/anti-patterns.md`
- 要落到页面、海报、图文区块结构时，读 `references/page-and-poster-patterns.md`

## Output Style

- 先判断，再修改
- 先指出偏差，再给方案
- 用克制、清晰、有人味的语言，不要浮夸
- 不要把最终输出写成堆砌术语的设计黑话
