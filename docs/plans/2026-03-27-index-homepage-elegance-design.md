# 良禽佳木规范首页优雅化改版设计稿

## 背景

当前首页已经具备完整的信息结构、品牌 token 入口和组件规范内容，但视觉重心过于平均。页面从首屏到正文大量依赖相似权重的 section 与卡片容器，导致阅读节奏偏平，优雅感更多来自配色与字体，而不是来自版式主次、展陈层级与策展式叙事。

这一轮改版聚焦首页 [index.html](/Users/admin/Nutstore%20Files/我的坚果云/CODE/project/liangqin-brand-design/index.html)，目标是在不破坏规范站清晰度的前提下，把它升级为更像“品牌设计系统入口”的页面，而不是普通 style guide 文档。

## 已确认方向

- 改版对象：`index.html`
- 风格方向：`混合型`
- 主要优化维度：`版式节奏 + 信息层级`
- 主题策略：默认跟随系统，同时提供显式切换按钮
- 主题模式：`浅色 / 深色 / 跟随系统`

## 设计判断

### 1. 首页要从“平均的说明页”变成“有主次的入口页”

现有首页的优势是完整、稳定、信息清楚，但从 Hero 到各 section 的视觉权重差距较小。用户很快能知道“这里有很多规范内容”，但不容易在前 10 秒感受到品牌方法、入口优先级和最值得先看的内容。

因此，本轮不以“再加更多精致卡片”为方向，而是通过拆分内容层级、减少平均权重、增强阅读起伏来提升优雅感。

### 2. 信息架构保留，但阅读层次改为四段递进

首页保留左侧目录与现有大章节，但右侧阅读逻辑调整为四层：

1. `Editorial Intro`
   作为品牌前言与系统入口，回答“这是什么、服务谁、如何使用”。
2. `Brand Assets / Foundations`
   作为识别系统与设计基础的主展陈层，强调 Logo、色彩、排版等高价值资产。
3. `Components / Principles`
   作为可执行规则层，把组件从展示样本转为设计判断入口。
4. `Appendix / Links`
   作为索引出口层，承接 schema、examples、下载入口等次级资源。

### 3. 视觉优雅感来自“不平均”

优雅感的主要来源不是更强的装饰，而是明确主次：

- 关键区块允许更大留白、更长阅读宽度和更少边框
- 辅助区块更轻、更短、更像注释与索引
- 某些 section 会采用“大主块 + 小辅助块”的非均匀布局
- Appendix 和资源入口将刻意降低视觉存在感，避免与品牌主张抢权重

### 4. 深色主题要像“夜间展厅”，不是暗黑 SaaS

既然项目设计上下文已经明确要求支持深色主题，首页需要补齐一套与品牌气质一致的深色版本。深色模式不应采用纯黑、霓虹、高反差科技风，而应延续当前品牌的暖灰、taupe、柔和分界和低噪声特征。

原则如下：

- 背景用暖炭黑与深褐灰，而不是纯黑
- 卡片通过层差与边界区分，而不是重阴影
- accent 继续克制，仅作为锚点与选择反馈
- 标题、正文、分隔线维持温润可信的基调

### 5. 主题切换采用三态策略

为了兼顾日常浏览与视觉对比，主题不只支持系统自动，而是提供三个模式：

- `系统`
- `浅色`
- `深色`

用户选择需要本地持久化。默认首次访问跟随系统，手动切换后优先遵循显式选择。

## 具体改版策略

### 首屏

- 把当前 Overview 改为更像品牌前言的双栏结构
- 左侧呈现品牌方法、服务对象与系统用途
- 右侧提供高权重系统入口，如 DNA、Tokens、Skill
- 用更强的排版和节奏建立第一个视觉重心

### Brand Assets / Foundations

- 提升 Logo、Color、Typography 等高价值内容的展陈感
- 用更大的主展区承载资产示意，用更轻的辅助块承载规则说明
- 减少“所有块都同样重要”的感觉

### Components

- 将组件 section 从“样例展示”转向“设计原则 + 样例支撑”
- 强化为什么这样设计，而不仅是展示长什么样
- 让用户能直接感受到这套系统背后的判断方法

### Appendix

- 作为资源出口层，视觉权重低于前文主体
- 卡片更轻、更理性，更像索引和链接目录

## 成功标准

改版后需要达到：

1. 用户 10 秒内能理解这是什么、服务谁、从哪里开始看
2. 页面有明显主次与节奏，不再是连续均匀卡片流
3. 品牌气质更接近高端中文家居品牌的系统入口
4. 深色模式依旧温润、克制、可信，不转向科技暗黑风
5. 主题切换可快速用于视觉对比，并记住用户选择
