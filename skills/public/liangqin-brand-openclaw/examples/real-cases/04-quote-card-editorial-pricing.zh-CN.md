# 真实案例 04｜图文报价体报价卡蓝图

## 场景

- 页面类型：图文报价体 / 报价卡长图
- 示例任务：将一段正式报价整理成可发送给客户的单张或少量图片
- 页面目标：让客户先看懂产品与总价，再理解确认条件与必要计价依据

## 输入摘要

### 报价资料

- 报价类型：正式报价 / 参考报价汇总
- 内容来源：结构化报价 payload 或整理好的正式报价文本
- 核心字段：产品名、总价、已确认条件、关键计价依据、补充说明
- 输出形式：手机优先的 JPG 长图

### 品牌要求

- 不做促销海报感
- 不做 Excel 截图感
- 金额必须是第一视觉焦点
- 完整计算过程仍以文字报价为准

## 推荐信息顺序

1. `quote-hero`
2. `confirmed-context`
3. `pricing-groups`
4. `notes`
5. `footer-hint`

## 设计判断

- 产品名与总价必须先被看到，不能让公式先抢眼
- 单品报价优先按“基础计价 / 调整项 / 最终报价”分组
- 多品报价优先展示总价、产品列表与必要说明，不强塞全部公式
- 图片只做气质托底，不做主视觉炫技
- 内部规则提示、系统路径和工具错误都不能出现在客户图卡里

## 结果特征

- 图卡看起来像一张可信、温润、专业的正式报价卡
- 阅读顺序自然落在“总价 → 条件 → 依据 → 说明”
- 适合会话发送、客户转发与快速确认
- 与良禽佳木既有 DNA 保持一致，不偏成商城促销图或技术报表
