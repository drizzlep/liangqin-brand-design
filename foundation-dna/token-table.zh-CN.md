# 中文化 Token 建议表

> 当前版本：`v2`  
> 适用范围：中文高端家居品牌官网、案例页、产品详情页、咨询入口、组件系统

## 1. 品牌原则

| 原则 | 说明 | 落地要求 |
|---|---|---|
| 高级但不疏离 | 高端感来自材质、比例、留白 | 不用强营销色块和过度戏剧化文案 |
| 温润且安静 | 让用户感到稳定、可信、可久看 | 颜色以暖白、石灰、烟灰、炭黑为主 |
| 秩序清晰 | 信息必须一眼能懂 | 栅格、标题、辅助信息层级明确 |
| 强调触感 | 通过摄影和文案传递木、石、布、皮革 | 少图形噪音，多真实材质画面 |
| 中文优先可读 | 中文阅读体验高于纯西式美感 | 正文字号、行高、标签长度优先保障 |

| `--dna-font-mono` | `SF Mono / Menlo / Consolas` | 数据、编号、工程信息 |

| `--dna-size-overline` | `12px` | 分组标签 / 小标题 |

### 中文排版默认规则

- 正文默认 `17px / 1.82`，不要小于 `16px / 1.7`
- 中文大标题默认优先使用 Swei Sugar（狮尾宋体）
- 导航、按钮、正文、摘要优先使用 OPPO Sans 4.0 这类稳定 sans
- 中文标题可使用 serif，但必须保持字重足够、字间距不过分压缩
- 连续多行中文标题时，优先控制宽度，不要过度依赖字距制造高级感

### 布局原则

- 默认采用 **12 栏桌面 / 8 栏平板 / 4 栏手机**
- 首页 Hero 可局部居中，但正文内容应更多依赖严格网格
- 产品、案例、方法论页面优先使用统一版心，减少任意漂浮布局

## 5. 形状、边框、层次、动效

| Token | 值 | 用途 |
|---|---:|---|
| `--dna-radius-sm` | `2px` | 图片蒙层 / 微边角 |
| `--dna-radius-md` | `6px` | 表单 / 轻卡片 |
| `--dna-radius-lg` | `12px` | 浮层 / 柔和卡片 |
| `--dna-radius-pill` | `999px` | 胶囊标签 / 部分按钮 |
| `--dna-shadow-low` | `0 8px 24px rgba(31,29,27,.04)` | 轻层级 |
| `--dna-shadow-medium` | `0 18px 44px rgba(31,29,27,.08)` | 浮层 |
| `--dna-shadow-high` | `0 28px 72px rgba(31,29,27,.12)` | 强强调层 |
| `--dna-ease-standard` | `cubic-bezier(0.22, 1, 0.36, 1)` | 默认缓动 |
| `--dna-duration-micro` | `140ms` | hover / press |
| `--dna-duration-normal` | `280ms` | 淡入 / 展开 |
| `--dna-duration-macro` | `520ms` | 首屏切换 / 图片揭示 |

### 动效原则

- 只允许：淡入、轻位移、弱视差、图片轻揭示
- 不建议：粒子、流体、磁吸鼠标、复杂 3D、shader 扭曲
- 最理想的效果是“用户觉得页面很顺，但不明显意识到有特效”

## 6. 中文客户默认交互规则

### 导航
- 一级导航少而准，命名完整，不做英文式极短命名
- 移动端优先保证路径理解，不追求花哨开合

### 按钮
- 主按钮宜深底浅字，次按钮宜浅底描边
- CTA 文案以“查看 / 了解 / 预约 / 获取”为主，避免命令式压迫语气

### 表单
- 标签始终显式可见，不仅靠 placeholder
- 帮助说明、错误信息、成功反馈要有稳定层级
- 中文字段长度预留要足，不做英文宽度假设

### 内容模块
- 图片优先于图形装饰
- 数据、参数、材质说明要有明确节奏和对齐
- 方法论与品牌叙事页可更理性排版，但不能失去温度

## 7. 推荐组件继承关系

- `Button`：继承色彩层级与礼貌 CTA 语气
- `Navigation`：继承严谨栅格与低噪声交互
- `Card`：继承大留白和弱边界，不做电商促销卡感
- `Form`：继承中文优先可读与说明性文案风格
- `Content Module`：继承摄影优先、文本克制、结构清晰

## 8. 验证清单

在后续页面和组件落地时，用下面 5 项验证：

1. 首页 Hero 是否同时体现高级感、秩序感与中文可读性
2. 产品 / 案例详情是否能统一容纳图片、参数、说明与 CTA
3. 品牌故事 / 方法论页是否兼具理性编排与温度
4. 导航 / 表单是否比纯欧美参考更适合中文用户理解
5. 移动端是否仍保有留白与节奏，而不是挤压成普通资讯页

## 9. 响应式与移动端规则（v2 新增）

### 移动端默认原则

- 首屏可更集中，但不能拥挤；主标题与摘要必须仍然“像品牌在说话”
- 导航可横向滑动或折叠，但栏目命名不能为了省空间而失真
- 主要 CTA 建议满宽展示，减少误触与犹豫
- 案例、FAQ、服务步骤转单列后，仍需保留清晰的标题—摘要—动作节奏

### 响应式建议表

| 维度 | 桌面 | 平板 | 手机 |
|---|---|---|---|
| 栅格 | 12 栏 | 8 栏 | 1-4 栏内容组织 |
| 区块节奏 | 96-128px | 72-96px | 64px 左右 |
| 正文建议 | 17px | 16-17px | 16px |
| CTA 布局 | 并列主次按钮 | 主次并列或折行 | 主操作优先满宽 |
| 导航策略 | 全量可见 | 精简但完整 | 横滑 / 折叠，命名完整 |

## 10. 内容模块规则（v2 新增）

| 模块 | 作用 | 默认组成 |
|---|---|---|
| `material-note` | 解释材质与触感 | 标题、图片、正文、辅助事实 |
| `service-path` | 说明服务流程 | 步骤号、标题、说明、可选 CTA |
| `case-summary` | 快速呈现场景差异 | 类型标签、标题、摘要 |
| `consultation-promise` | 建立咨询信任感 | kicker、标题、承诺列表、次级动作 |

### 内容与转化规则

- CTA 语气始终是“邀请”，不是“催促”
- 表单提交前要先解释：提交后会发生什么、客户会获得什么
- 信任感优先来自案例结构、材质说明、流程透明，而不是强刺激促销词
- 中英双语共存时，英文只作为辅助层，不应反向主导中文层级
